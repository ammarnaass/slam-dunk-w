# دليل الإدارة - موقع سلام دانك

## نظرة عامة
يستخدم هذا الموقع ملفات JSON لإدارة الحلقات والشخصيات. لا حاجة لقاعدة بيانات معقدة - كل ما تحتاجه هو تعديل ملفات JSON.

## إضافة حلقة جديدة

### الخطوة 1: رفع الفيديو على Mega
1. قم بإنشاء حساب على [Mega.nz](https://mega.nz)
2. ارفع ملف الفيديو إلى حساب Mega الخاص بك
3. انقر بزر الماوس الأيمن على الملف واختر "Get link"
4. احفظ رابط المشاركة (مثال: `https://mega.nz/file/XXXXXXX#YYYYYY`)

### الخطوة 2: تحديث ملف episodes.json
افتح الملف: `src/data/episodes.json`

أضف كائن جديد للحلقة بالشكل التالي:

```json
{
  "id": "4",
  "title": "عنوان الحلقة بالعربية",
  "description": "وصف تفصيلي للحلقة",
  "season": 1,
  "episode_number": 4,
  "thumbnail": "رابط صورة الحلقة",
  "duration": "24:15",
  "mega_link": "https://mega.nz/file/XXXXX#YYYYY",
  "video_url": ""
}
```

### شرح الحقول:
- **id**: معرف فريد للحلقة (رقم أو نص)
- **title**: عنوان الحلقة
- **description**: وصف قصة الحلقة
- **season**: رقم الموسم
- **episode_number**: رقم الحلقة
- **thumbnail**: رابط صورة بحجم 800×450 تقريباً
- **duration**: مدة الحلقة بصيغة "MM:SS"
- **mega_link**: رابط المشاركة من Mega
- **video_url**: اتركه فارغاً إذا كنت تستخدم Mega فقط، أو ضع رابط MP4 مباشر

### ملاحظة مهمة حول التشغيل المباشر
- إذا كان `video_url` فارغاً، سيظهر زر "مشاهدة على Mega" بدلاً من المشغل
- إذا أردت تشغيلاً مباشراً، ضع رابط MP4 مباشر في `video_url`

## إضافة شخصية جديدة

### الخطوة 1: تحضير صورة الشخصية
- حجم موصى به: 600×800 بكسل (نسبة 3:4)
- يمكنك استخدام خدمات مثل Imgur أو Cloudinary لاستضافة الصور

### الخطوة 2: تحديث ملف characters.json
افتح الملف: `src/data/characters.json`

أضف كائن جديد للشخصية:

```json
{
  "id": "mitsui-hisashi",
  "name_ar": "ميتسوي هيساشي",
  "name_en": "Hisashi Mitsui",
  "name_jp": "三井 寿",
  "role": "Shooting Guard",
  "height": "184 cm",
  "weight": "70 kg",
  "team": "Shohoku",
  "number": 14,
  "bio": "لاعب الهداف السابق في المرحلة الإعدادية. عاد لكرة السلة بعد فترة غياب.",
  "image": "رابط_صورة_الشخصية"
}
```

## النشر والتحديث

### التطوير المحلي
```bash
npm run dev
```
- يفتح الموقع على: http://localhost:3000
- التغييرات على ملفات JSON تتطلب إعادة تحميل الصفحة

### البناء للإنتاج
```bash
npm run build
npm start
```

### النشر على Vercel
1. ادفع التغييرات إلى Git
2. سيتم النشر تلقائياً إذا كنت متصلاً بـ Vercel
3. أو استخدم الأمر: `vercel --prod`

## نصائح مهمة

### الصور
- استخدم خدمات استضافة صور موثوقة
- تأكد من أن الروابط تدعم HTTPS
- الحجم الموصى به: أقل من 500 كيلوبايت للصورة

### روابط Mega
- تأكد من أن رابط المشاركة "Public"
- لا تشارك رابط مجلد كامل، استخدم رابط الملف المباشر
- الروابط لا تنتهي طالما الملف موجود في حسابك

### SEO
- اكتب عناوين ووصوفات واضحة
- استخدم كلمات مفتاحية ذات صلة بالأنمي
- سيتم إنشاء Sitemap تلقائياً

## استكشاف الأخطاء

### الحلقة لا تظهر
- تأكد من صحة JSON syntax
- تحقق من عدم وجود فاصلة زائدة في آخر عنصر
- أعد بناء المشروع: `npm run build`

### الصور لا تُحمّل
- تأكد من أن رابط الصورة يعمل في المتصفح
- تحقق من أن الاستضافة تسمح بـ hotlinking

### المشغل لا يعمل
- إذا كان video_url فارغاً، هذا متوقع (سيظهر زر Mega)
- إذا كان لديك رابط MP4، تأكد أنه direct link وليس صفحة ويب
